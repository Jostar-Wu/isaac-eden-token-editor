# 伊甸次数修改器设计规格

## 1. 项目目标

制作一款可在 GitHub 免费分享的 Windows 中文桌面程序，帮助《以撒的结合》玩家安全修改伊甸使用次数（Eden Tokens）。第一版只修改这一项数据，兼容 Repentance 与 Repentance+ 存档。

程序提供两条独立流程：

- 自动模式：寻找 Steam 存档、列出存档位、备份原文件、修改并安全写回。
- 手动模式：用户选择一个 `.dat` 文件，程序修改后导出新文件，并提供明确的替换步骤；原文件不变。

所有存档处理均在用户电脑本地完成。程序不上传存档，不需要账户，不依赖网络服务。

## 2. 范围

### 包含

- Windows 10 与 Windows 11。
- Repentance 的 `rep_persistentgamedata1.dat`、`rep_persistentgamedata2.dat`、`rep_persistentgamedata3.dat`。
- Repentance+ 的 `rep+persistentgamedata1.dat`、`rep+persistentgamedata2.dat`、`rep+persistentgamedata3.dat`。
- 自动发现 Steam 安装目录及 `userdata/<SteamID>/250900/remote`。
- 手动选择任意位置的受支持存档。
- 读取并显示版本、存档位、文件路径和当前伊甸次数。
- 接受 0 至 4,294,967,295 范围内的整数。
- 修改伊甸次数并重算游戏使用的 CRC。
- 自动模式写回前备份，写回后重新读取验证。
- 手动模式导出同名文件到用户选择的新目录。
- 中文界面、中文 README、GitHub Release 中的免安装单文件程序。

### 不包含

- Afterbirth+、Rebirth 或主机版存档。
- 修改捐款机、连胜、死亡次数、成就、角色标记或其他数据。
- 自动操作 Steam 云同步界面。
- 在游戏运行时修改存档。
- 上传、收集或分析用户存档。
- 自动更新、遥测、账号系统或联网后端。

## 3. 技术方案

采用 C#、.NET 8 和 WPF。发布目标为 `win-x64` 自包含单文件应用，用户无需预装 .NET。

代码按职责拆分为以下模块：

- `SaveDiscovery`：读取 Steam 注册表、解析 Steam library 和 userdata，发现所有候选存档。
- `SaveParser`：验证文件头、版本、区段布局、文件长度及 CRC，定位伊甸次数字段。
- `EdenTokenEditor`：在内存副本中写入目标数值并更新 CRC，不直接负责磁盘写入。
- `BackupService`：为自动模式创建带时间戳的备份目录并复制原文件。
- `SaveWriter`：使用同目录临时文件写入、验证后原子替换，避免半写入文件。
- `GameProcessGuard`：检查 `isaac-ng.exe` 是否运行。
- `MainViewModel`：协调自动扫描、手动导入、修改、导出、写回和界面状态。
- `HelpContent`：提供路径说明、替换步骤、Steam 云冲突说明和错误解决建议。

解析与修改核心不依赖 WPF，便于单元测试，也允许未来添加其他界面而不改变存档逻辑。

## 4. 存档发现

自动发现流程：

1. 从 `HKCU\\Software\\Valve\\Steam` 读取 Steam 安装路径。
2. 搜索 Steam 根目录下的 `userdata/*/250900/remote`。
3. 在每个候选目录中寻找 Repentance 与 Repentance+ 的三个持久存档文件。
4. 逐个进行只读解析；无效文件显示原因，不作为可修改项。
5. 若发现多个 Steam 账户，按 Steam userdata ID 和目录分组展示，由用户明确选择。

程序同时提供“选择 `.dat` 文件”按钮与拖放区域。手动模式不要求文件位于 Steam 目录，但仍必须通过完整格式验证。

## 5. 存档解析与修改

程序不得依赖某个历史版本的绝对固定文件偏移。它从存档区段表计算统计区起点，再依据受支持版本的统计字段布局定位伊甸次数。

解析顺序：

1. 验证文件最小长度与 `ISAACNGSAVE` 文件头。
2. 读取版本字节并映射到 Repentance 或 Repentance+。
3. 解析区段描述，使用受支持版本对应的条目长度计算各区段起点。
4. 检查所有计算范围均在文件边界内，且区段数量不会导致整数溢出。
5. 计算完整 CRC 并与文件尾四字节的小端值严格比较。
6. 从统计区读取四字节无符号伊甸次数。

修改过程仅对内存副本执行：写入新的四字节无符号整数，重算 CRC，将新 CRC 写入文件尾，并重新解析内存副本确认目标值和 CRC 均正确。

格式研究可引用公开项目，但实现代码应独立编写；未经兼容许可证允许，不直接复制第三方源代码。

## 6. 用户界面

应用使用单窗口布局，顶部持续显示“所有处理均在本机完成”和游戏进程状态。

初始区域包含：

- 推荐操作“自动寻找存档”。
- 次要操作“选择 `.dat` 文件”。
- 简短的受支持版本说明。

自动扫描结果以表格展示：Steam userdata ID、存档位、文件名、版本、当前伊甸次数和完整路径。用户选中一行后进入修改区。

修改区包含：

- 当前值。
- 只接受整数的目标值输入框，默认不擅自填入极大值；可提供 `100000` 快捷按钮。
- 自动发现的文件只显示“备份并写回”，完成备份、修改、写回和落盘验证的全自动流程。
- 手动选择的文件只显示“仅导出新文件”。

长时间操作期间禁用重复点击。每个结果都以明确的成功或失败状态结束，不使用仅包含“发生错误”的模糊提示。

## 7. 自动模式写回

写回前必须依次完成：

1. 确认文件仍存在，且路径与用户选择一致。
2. 确认 `isaac-ng.exe` 未运行。
3. 再次读取源文件并验证 CRC；若文件自扫描后发生变化，要求用户重新扫描。
4. 在 `%USERPROFILE%\\Documents\\IsaacEdenEditorBackups\\yyyyMMdd-HHmmss` 创建备份，并保留原文件名。
5. 重新读取备份并比较字节哈希，确认备份完整。
6. 在源文件同目录写入临时文件并验证。
7. 原子替换目标文件；替换失败时保留源文件和备份。
8. 重新读取目标文件，确认版本、文件长度、目标数值和 CRC。

完成页显示修改后的数值、实际存档路径、备份目录、“打开备份文件夹”按钮以及 Steam 云冲突说明。

## 8. 手动模式导出与替换指导

手动模式永不覆盖导入的源文件。用户选择一个空目录或新建目录后，程序以游戏要求的准确原文件名导出修改结果，例如 `rep+persistentgamedata1.dat`。

若目标目录已存在同名文件，程序要求用户明确确认或选择其他目录。

导出完成页提供“打开导出文件夹”，并显示：

1. 完全关闭游戏。
2. 找到软件扫描结果或帮助页中显示的 Steam 存档目录。
3. 备份目录中的原同名文件。
4. 将导出的文件复制到该目录并覆盖同名文件。
5. 启动游戏检查伊甸次数。
6. 若 Steam 提示云冲突，选择修改时间较新的本地文件；不确定时先取消同步并恢复备份。

## 9. 错误处理

以下情况禁止修改或写回，并显示具体解决建议：

- 文件头、版本或区段结构不受支持。
- 文件 CRC 不匹配。
- 文件已被修改、删除、锁定或权限不足。
- 游戏进程正在运行。
- 输入为空、包含非数字字符或超出无符号 32 位范围。
- 自动扫描发现多个账户但用户未选择。
- 备份复制或哈希验证失败。
- 临时文件验证失败、原子替换失败或写回后验证失败。

异常信息不得显示堆栈、注册表隐私信息或无关系统数据。诊断日志默认只保存在内存；用户主动选择“复制诊断信息”时，内容仅包括应用版本、操作阶段、错误类型、受支持版本标识和脱敏路径。

## 10. 测试策略

### 单元测试

- Repentance 与 Repentance+ 的有效合成测试夹具均能正确读取伊甸次数。
- 三个存档位文件名均能识别。
- 最小值 0、常用值 100000 和最大值 4,294,967,295 可往返写入。
- 修改后除目标字段与 CRC 外，其余字节保持不变。
- 原文件 CRC 正确时通过，单字节损坏时拒绝。
- 截断文件、伪造区段数量、越界偏移和不支持版本均安全失败。
- 自动发现支持默认与非默认 Steam 安装路径、多 userdata 账户和无存档情况。

### 集成测试

- 自动模式确实先备份、后写临时文件、再替换，并能验证最终文件。
- 模拟备份失败、文件锁定、替换失败时原文件保持不变。
- 游戏进程守卫阻止写回，但仍允许只读扫描和手动导出。
- 手动导出保留准确文件名且不改变源文件。

### 发布验证

- 在干净的 Windows 10 与 Windows 11 环境运行自包含单文件程序。
- 使用项目内不含用户隐私的合成夹具进行端到端验证。
- 对 Release 中的 `.exe` 生成 SHA-256，并在发布说明中列出。

不得把用户真实存档提交到 Git 仓库或测试夹具中。

## 11. GitHub 发布内容

仓库暂定名 `isaac-eden-token-editor`，使用 MIT License，并明确声明项目与 Nicalis、Edmund McMillen 及《以撒的结合》官方无关联。

仓库包含：

- C# 源代码与测试。
- 中文 `README.md`，含截图、功能、支持范围、自动与手动教程、备份恢复和云冲突说明。
- `SECURITY.md`，说明程序离线工作及如何报告存档损坏问题。
- `LICENSE`。
- GitHub Actions：在 Windows 上构建、测试，并为带标签版本产出单文件 `.exe` 与 SHA-256。
- GitHub Release：免安装程序、校验值和变更说明。

第一版不申请代码签名证书。README 需解释未签名程序可能触发 Windows SmartScreen，并指导用户核对 GitHub Release 来源及 SHA-256，不建议用户关闭杀毒软件。

## 12. 完成标准

项目只有同时满足以下条件才算完成：

- 自动扫描和手动选择均可用。
- Repentance 与 Repentance+ 的有效存档可读取并修改。
- 自动模式在任何写回前建立并验证备份。
- 自动写回和手动导出结果均通过重新解析与 CRC 验证。
- 所有失败路径不会损坏或静默覆盖原文件。
- 测试通过，干净 Windows 环境可运行发布包。
- GitHub 仓库包含源码、许可证、中文教程、Release 程序和 SHA-256。
